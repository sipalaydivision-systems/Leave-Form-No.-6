External Entities (Users)

  • Employee — DepEd staff submitting leave
  • Administrative Officer (AO) — School-level approver
  • Human Resources (HR) — Division-level certifier
  • ASDS — Assistant Superintendent approver
  • SDS — Final approving authority
  • IT Administrator — System maintainer

External Systems

  • MailerSend — Transactional email delivery API
  • File System — Railway persistent volume (JSON stores)
  • Browser — Vanilla JS web client for all portals

Approval Chain

  • Step 1: Employee submits application
  • Step 2: AO reviews and forwards to HR
  • Step 3: HR certifies leave credits
  • Step 4: ASDS reviews and recommends
  • Step 5: SDS gives final approval
  • Any step can return or reject

Entity Summary

Description of each entity, its purpose, and key constraints.

EMPLOYEE_USER

  • DepEd employee accounts
  • PK: id (UUID)
  • UK: email (@deped.gov.ph), employeeNo
  • Password: bcrypt (12 rounds)
  • File: users.json

ADMIN_USER

  • AO / HR / ASDS / SDS / IT accounts
  • PK: id (UUID)
  • UK: email per portal file
  • role: ao | hr | asds | sds | it
  • Files: ao/hr/asds/sds/it-users.json

LEAVE_APPLICATION

  • Core workflow entity
  • PK: id (SDO Sipalay-NN-XXXX)
  • FK: employeeEmail → EMPLOYEE_USER
  • status: pending | approved | rejected | returned
  • File: applications.json

APPROVAL_HISTORY

  • Embedded in LEAVE_APPLICATION
  • One entry per approval stage action
  • Tracks portal, approver, action, timestamp
  • Immutable audit trail

LEAVE_CARD

  • Employee leave balance record
  • PK: employeeId (= email)
  • One card per employee
  • Annual quotas reset yearly (FL/SPL/WL)
  • File: leavecards.json

LEAVE_TRANSACTION

  • Embedded in LEAVE_CARD.transactions[]
  • type: ADD (accrual) | LESS (usage) | LAWOP
  • Running balance per transaction
  • Source tracks whether system or manual

LEAVE_USAGE_HISTORY

  • Embedded in LEAVE_CARD.leaveUsageHistory[]
  • Created on SDS final approval
  • Tracks balance snapshot after each usage
  • Links application to card deduction

CTO_RECORD

  • Compensatory Time-Off grants
  • Multiple records per employee (per Special Order)
  • LIFO deduction order
  • File: cto-records.json

PENDING_REGISTRATION

  • Registration requests awaiting IT approval
  • status: pending | approved | rejected
  • Approved → creates EMPLOYEE_USER or ADMIN_USER
  • File: pending-registrations.json

ACTIVITY_LOG

  • Immutable audit trail of all actions
  • Archived when exceeds 10,000 entries
  • Includes IP address, user agent
  • File: activity-logs.json

SYSTEM_STATE

  • Global singleton for accrual tracking
  • lastAccruedMonth prevents double-accrual
  • Updated after every accrual run
  • File: system-state.json
